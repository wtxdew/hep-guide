SSH Login Without Password

There are several ways to login to the server without a password.

SSH key authentication‚Äã

Âç±Èö™

For security, it is you must add a passphrase to the SSH key.

SSH key is the best and most convenient method for SSH authentication. With ssh-agent, you can easily implement ssh password-free login. Please refer to the GitHub document for a detailed setup: Generating a new SSH key and adding it to the ssh-agent - GitHub Docs

The only disadvantage of SSH key is that some servers (e.g. grid5, lxplus) do not allow SSH key authentication and must use a password.

2. The SSH_ASKPASS

If you must use a password login because the server manager requires it, this method is probably a good solution for you. You can just follow this site, it is clear enough. Say 'No' to sshpass. To prevent this site from dead, I'll explain how to set it up below.

Setup:‚Äã

This setup method is only for MacOS. If you are using a different operating system, you will need to find the setup instructions on your own.

  1. Store your password in Keychain Access.app which is the default app in MacOS.
For anybody don‚Äôt know how to do:
  1. Open the Keychain Access.app
  2. Press ‚åò + N to add a new password.
  3. Remember your itemName and accountName. They will be used later.

  1. Place the following code in the file ~/.local/bin/askpass (or any path you want).

    ~/.local/bin/askpass
    #!/usr/bin/env bash
    /usr/bin/security find-generic-password -a <accountName> -s <itemName> -w
    ÊèêÁ§∫

    This script works only for a single password. If you have multiple settings, you will need to make some changes.

  2. make it executable:

    $ chmod u+x ~/.local/bin/askpass

  3. install notty following the GitHub repo. You need to compile it by yourself.

    GitHub - pharaujo/notty: notty is a command line tool to detach a process from a tty.

  4. Create another script ~/.local/bin/myssh

    ~/.local/bin/myssh
    #!/usr/bin/env bash

    export DISPLAY=:99
    export SSH_ASKPASS="$HOME/.local/bin/askpass"

    notty ssh $@

  5. Then you can use myssh USER@HOSTNAME to log in ssh server without a password.

3. iTerm2 password manager

  • Using iTerm2 is an easy way, you don‚Äôt need to set up anything.

4. Termius

Termius is an SSH client that remembers your password. It doesn't use X11 forwarding, but you can use other drawing methods to get around it. It is basically free, and you can request GitHub education to get the full version.
